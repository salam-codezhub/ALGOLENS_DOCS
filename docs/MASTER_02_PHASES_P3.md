# GLOBAL RULES FOR EVERY PHASE

Before marking a phase complete, verify:

* Project builds successfully.
* TypeScript compiles without errors.
* ESLint passes.
* Prettier formatting applied.
* No duplicate code introduced.
* Imports are organized.
* No unused variables remain.
* No placeholder implementations remain.
* New code is documented.
* Existing functionality is not broken.
* All deliverables for the phase are complete.

If any of the above fail:

Do NOT mark the phase complete.

Fix the issues first.

Only then stop and wait for the next phase command.

---

# COMMUNICATION RULES

When a phase starts:

1. Restate the phase objective.
2. List deliverables.
3. Explain dependencies.
4. Generate all required files.
5. Explain any new libraries added.
6. Confirm build status.
7. Stop and wait.

Never continue to the next phase automatically.

AlgoLens is considered complete only when:

* All 40 phases are completed.
